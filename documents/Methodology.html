03

视频运动学指标计算（逐帧）

当前视频分支先对 17 个关键点做 median + Savitzky-Golay 预处理，并用预处理后的关键点重新计算 2D CoM；CoM 再做 centered moving average(5) 平滑。metric CSV 直接使用平滑后的 com 字段，不再二次平滑；随后每帧用两肩关键点距离换算缩放，计算摆长 l、CoM 位置、速度、自然频率 ω₀ 与外推重心 xCoM（倒立摆 + Hof 模型）。

视频 · compute_keypoints_smooth_video_metric.py

1
像素→米缩放（每帧独立，两肩宽参考 0.34 m）shoulder_width_px = ||left_shoulder − right_shoulder||ppm = shoulder_width_px / 0.34 m
2
摆长 l（重心离地高度；视频 y 轴向下，取更靠近地面的踝点）ground_y = max(left_ankle_y, right_ankle_y)l_m = (ground_y − com_y) / ppm
3
CoM 位置（向上为正）com_x_m = com_x_px / ppmcom_y_m_up = −com_y_px / ppm
4
速度（向后差分 × 帧率）v = Δr · fps
5
自然频率 ω₀ = √(g / l_m)
6
外推重心 xCoM xCoM = CoM + v / ω₀
Output
validation/metrics_keypoints_preprocessed/video_com_metric.csv
每帧：scale_method=shoulder_width、位移 / 速度 / l / ω₀ / xCoM（米制，向上为正）

07

Vicon 运动学指标计算（逐帧）

在 3D 毫米重心基础上换算为米，计算摆长 l、位移、速度、自然频率 ω₀ 与外推重心 xCoM。公式与视频侧同源，但单位与差分方式不同。

Vicon · validate_com_normalized.py

1
单位换算 com_m = com_mm / 1000
2
摆长 l（直接用 Z 高度，无需换算）l = com_z_m
3
位移（帧间差分，三维）Δr = com_m[i] − com_m[i−1]
4
速度（中心差分，更精确）v = np.gradient(com_m, t)
5
自然频率 ω₀ = √(g / com_z_m)
6
外推重心 xCoM（三维向量）xCoM = CoM + v / ω₀
Output
vicon_com_metric.csv
每帧：三维位移 / 速度 / l / ω₀ / xCoM（米制）
视频 vs Vicon 关键差异：视频是 2D（像素，当前用两肩距离 / 0.34 m 换算缩放）、速度用向后差分；Vicon 是 3D（毫米，直接读取）、速度用 np.gradient 中心差分（误差 O(Δt²)，更精确）。ω₀ 与 xCoM 公式完全相同。
08

相关性验证 （核心 · CoM 相关性 + 关键点相关性）

逐试验、逐轴（水平 x / 垂直 z）执行。当前 CoM 验证使用 keypoints 预处理且 CoM 平滑后的序列；两路采样率不同，先截重叠窗口，再把视频 CoM 线性插值到 Vicon 250 Hz 时间点，最后 detrend、z-score、xcorr 比较。

8.1 · CoM 相关性分析

输入 A
视频 CoM 序列
≈30 Hz
输入 B
Vicon CoM 序列
250 Hz
Step 1 · 截重叠窗口
取两路时间交集
t_min = max(start) ~ t_max = min(end)，截掉 >20% 记 warning
Step 2 · 插值（核心）
np.interp 线性插值
方向 视频 → Vicon 时间轴：在相邻两个视频帧之间插入/估计新的视频值，使这些估计点的时间与 Vicon 帧时间一致。对某个 Vicon 时刻 t_vicon，找到它左右相邻的两个视频时刻 t0, t1，再按时间比例估计该时刻的视频值：
video(t_vicon)=video0+(video1−video0)·(t_vicon−t0)/(t1−t0)
例如 Vicon 帧在 0.168 s，视频相邻帧在 0.167 s 和 0.200 s，则用这两个视频帧之间的比例估计 0.168 s 的视频 CoM。这样处理后，视频序列被补到 Vicon 对应时间点，两路信号才能在同一时刻逐点比较 r / nRMSE / xcorr。
Step 3 · 符号对齐
翻转 Vicon Y、Z
vicon_Y → −vicon_Y，vicon_Z → −vicon_Z
Step 4 · 线性去趋势
scipy.signal.detrend
最小二乘拟合直线后减去，去除步行整体漂移，只留步频节律
Step 5 · Z-score 归一化
(x − μ) / σ
消除量纲（像素 vs 毫米），只比较形状与节律
Step 6 · 输出
Pearson r · nRMSE · xcorr
写入 validation/com_keypoints_preprocessed/com_correlation.csv + com_z-score_detrend_xcorr/ lag 对齐叠加图

输出指标

Pearson r
线性相关系数 [−1,1]。输入已 Z-score，等价于逐点内积均值；附双尾 p 值。
nRMSE
归一化均方根误差。因输入已 Z-score（单位=标准差），RMSE 本身即归一化。0=一致，1=差一个标准差。
xcorr peak_r
滑动互相关峰值 = 最优时移下的最大相关，排除同步残差后的相关上限。
xcorr lag_frames / lag_ms
最优时移。lag>0 表 B 超前 A；lag<0 表 Vicon 超前视频，量化残余时间偏差。
n_frames
重叠窗口内参与计算的 Vicon 时间轴采样点数（因为当前插值方向是 video → Vicon）。
warning
时间重叠裁剪比例 >20% 时的提示标记。
